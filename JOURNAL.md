- 2026-01-07. Deep-dive analysis reveals Matomo counts 2.49x more GPS page views than Metabase server logs (after Dec 15), not just 2.21x more users. Key finding: ratio ACTIONS (22.65x) >> ratio USERS (2.21x), proving the issue is not user identification but page view counting. Example Dec 23: 795 Matomo hits vs 74 Metabase logs (10.74x) for /gps/groups/list alone. Root cause: client-side tracking (Matomo JS) fires multiple times per server request (page reloads, browser back/forward, prefetch). Zero anonymous users found. Recommendation: use Matomo events (category "gps") instead of page views, or Metabase logs as ground truth. Report: reports/2026-01-07-gps-matomo-metabase-decalage-actions.md.
- 2026-01-07. Documented root cause of Matomo vs Metabase GPS discrepancy: Matomo counts visits (any session containing a GPS page, segment pageUrl=@/gps/) while Metabase counts actions (user_id with logged server events). Ratio increased from 1.69x to 2.21x after Dec 15, 2025, explained by: (1) more exploratory traffic after sidebar link change (Nov 25), (2) ~50% of users generate only 1 log (brief consultations), (3) segment includes all pages in GPS sessions (dashboard, etc.). Created scripts/gps_matomo_vs_metabase_comparison.py for daily comparison. Report: reports/2026-01-07-gps-matomo-metabase-decalage.md. Key learning: the two metrics measure different but complementary things (discoverability vs engagement).
